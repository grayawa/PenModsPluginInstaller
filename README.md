# PenMods Plugin Installer

PenMods Plugin Installer is a native PenMods plugin for browsing the PenMods
Plugin Index, preparing plugin installs, tracking install state in SQLite, and
providing a small terminal installer for manual package imports.

The project currently contains both parts:

- A QML + native Qt plugin UI loaded by PenMods.
- A standalone `penmods-plugin` command-line installer for on-device manual
  installs.

## Status

Implemented:

- Registry fetching and local registry cache.
- Plugin install planning.
- Dependency and capability preflight checks before queueing installs.
- Missing required dependency queue records.
- Restricted or Telegram-style distribution handoff state.
- Core update planning for `/userdata/PenMods/libPenMods.so`.
- SQLite state tables for registry cache, installed plugins, install queue,
  dependency queue, and core updates.
- Standalone `penmods-plugin` CLI for installing a local plugin folder or zip.
- GitHub Actions CI for the Qt plugin, package layout, and aarch64 CLI build.

Not implemented yet:

- Automatic package download.
- Archive extraction inside the Qt backend.
- Metadata verification inside the queued installer worker.
- Actual queued install execution.
- Actual core `libPenMods.so` copy and restart handoff.

## Layout

- Managed plugins: `/userdisk/PenMods/plugins`
- PenMods core library: `/userdata/PenMods/libPenMods.so`
- Installer local data: `/userdisk/PenMods/plugins/plugin_installer/data`
- Registry source: `plugins.json` generated by PenMods Plugin Index

## Build

Install build dependencies on Arch or WSL Arch:

```bash
sudo pacman -Sy --needed base-devel xmake qt5-base qt5-declarative
```

Build the Qt plugin and host-development CLI:

```bash
xmake f -m debug
xmake
```

Build the device-compatible aarch64 CLI with the same glibc 2.27 toolchain used
by PenMods:

```bash
bash scripts/build-cli-aarch64.sh debug
```

Package the plugin:

```bash
bash scripts/package.sh debug
```

The packaged plugin is written to:

```text
package/plugin_installer
```

`package.sh` prefers `build/linux/arm64-v8a/debug/penmods-plugin` when it
exists, so release packages include the device-compatible CLI rather than the
host-development binary.

## Manual CLI

The package includes `bin/penmods-plugin`, which can be added to `PATH` on the
device:

```bash
penmods-plugin inspect ./plugin.zip
penmods-plugin inspect ./plugin-folder
penmods-plugin install ./plugin.zip
penmods-plugin install ./plugin-folder
penmods-plugin list
```

The CLI reads `metadata.json`, requires a plugin `id`, installs into
`/userdisk/PenMods/plugins/<sanitized-id>`, and uses `.new` / `.bak`
replacement paths so a failed install can roll back without overwriting the
existing plugin.

The device build is an aarch64 ELF and currently only requires `GLIBC_2.17`,
which is compatible with the PenMods glibc 2.27 environment.

## Native Plugin Shape

The plugin follows the current PenMods native plugin layout:

- `metadata.json`
- `main.qml`
- `libplugin_installer.so`

`libplugin_installer.so` exports `extern "C" void init_plugin()` and registers
`InstallerBackend` as `PenMods.PluginInstaller 1.0` for QML.

## Seeded IDs

The SQLite schema seeds two system entries:

- `lyrecoul.penmods`
- `com.penmods.plugininstaller`

These IDs are intended to match both the registry IDs and native
`metadata.json.id` values.

## CI

GitHub Actions builds and checks:

- Qt plugin shared library.
- Host-development CLI.
- aarch64 glibc 2.27-compatible CLI.
- Packaged plugin directory.
- `bin/penmods-plugin` architecture and GLIBC symbol version.

The packaged directory is uploaded as a workflow artifact.
